# AX 행사·교육 기획 보고서 하네스 (haness_ax_planning)

AX(AI Transformation) 포럼·워크샵·교육 등 **행사·교육 프로그램의 기획 보고서 작성에 특화된 Claude Code 하네스**.

Claude Code가 업계 최고 수준의 AX 컨설턴트로서 에이전트 팀을 조율하여, 리서치부터 개조식 기획 보고서 집필, 상급자 예상 질의 대응, 한글(hwpx) 파일 변환까지 한 번에 수행한다.

---

## 1. 하네스 주제

- **대상 사용자**: 기업에서 AX 추진 업무를 담당하는 실무자
- **해결하는 문제**: AX 포럼·워크샵·교육을 기획할 때, 상급자(임원·기관장) 결재를 통과할 수 있는 수준의 기획 보고서를 빠르게 작성
- **산출물 특징**:
  - 개조식(항목 나열식) + 공공기관 어투(「~함」, 「~임」, 「~하고자 함」)
  - 예산·일정·인력 근거를 갖춘 실행 가능한 계획
  - 상급자 예상 질의 Q&A 사전 준비
  - 보고서와 별도로 **작성 의미 해설**을 제공하여 담당자가 스스로 보고서를 쓸 수 있도록 학습 지원
  - 한글 양식(hwpx) 제공 시 최종본을 hwpx 파일로 변환

## 2. 구성 목적

| 목적 | 구현 방식 |
|------|----------|
| 기획 품질 확보 | 리서치 → 전략 → 도식 → 집필 → 검증의 5단계 분업 (에이전트 5명) |
| 결재 통과율 제고 | 상급자 관점 검증 에이전트가 예상 질의 Q&A와 실행가능성을 별도 점검 |
| 문체 일관성 | 개조식·공공기관 어투 규칙을 스킬로 표준화하여 모든 산출물에 적용 |
| 담당자 역량 성장 | 보고서의 각 절을 "왜 이렇게 썼는지" 해설하는 학습 자료 병행 산출 |
| 실무 포맷 대응 | 한글(hwpx) 양식 자동 채움 스킬로 기관 표준 서식에 맞춘 최종본 산출 |

## 3. 전체 구조

```
.claude/
├── CLAUDE.md                       — 페르소나·절대 원칙 선언 (모든 에이전트 공통)
├── agents/                         — 에이전트 팀 (5명)
│   ├── data-collector.md           — AX 트렌드·벤치마크 리서처
│   │                                 (최신 AX 트렌드, 유사 행사 사례, 강사 후보, 운영 참고자료 수집)
│   ├── analyst.md                  — 기획 전략가
│   │                                 (대상 분석, 프로그램 설계, 예산·일정·인력 실행가능성 검토, 리스크·Plan B)
│   ├── visualizer.md               — 기획서 도식 설계
│   │                                 (추진체계도, 프로그램 시간표, 예산표, 추진 일정표 — 한글 재현 가능한 표 중심)
│   ├── report-writer.md            — 기획 보고서 집필
│   │                                 (개조식·공공기관 어투 본문 + 절별 작성 의미 해설)
│   └── executive-summarizer.md     — 상급자 관점 검증(QA)
│                                     (예상 질의 Q&A, 실행가능성·정합성·어투 준수 교차 검증)
└── skills/                         — 스킬 (4개)
    ├── ax-planning-report/         — 오케스트레이터
    │                                 (팀 조율, 워크플로우, 실행 모드 판단, hwpx 변환 연계)
    ├── public-report-style/        — 개조식·공공기관 어투 작성 가이드
    │                                 (기호 체계 □→○→-→·, 어미 규칙, 서술형→개조식 변환 사전)
    ├── executive-qa-patterns/      — 상급자 예상 질의·실행가능성 점검 패턴
    │                                 (질의 5대 축, 행사 유형별 단골 질의, BSTR 점검, 리드타임 기준)
    └── hwpx-autofill-conversion/   — hwpx 양식 자동 채움 변환
                                      (hwpx ZIP·XML 구조 분석, 서식 보존 텍스트 채움, 재압축)
```

### 워크플로우

```
사용자 요청 (행사 유형·목적·대상·규모·예산·시기 [+ hwpx 양식])
        │
Phase 1  오케스트레이터: 요구사항 정리 → _workspace/00_input.md
        │
Phase 2  에이전트 팀 실행
        │   ① data-collector  → 01_research.md          (트렌드·벤치마크·강사 후보)
        │   ② analyst         → 02_planning_strategy.md (프로그램 설계·실행가능성·리스크)
        │   ③ visualizer      → 03_visual_elements.md   (체계도·시간표·예산표·일정표)  ─┐ 병렬
        │   ③ report-writer   → 04_planning_report.md   (개조식 보고서)                ─┘
        │                     → 05_report_commentary.md (작성 의미 해설)
        │   ④ executive-summarizer → 06_executive_qa.md (예상 질의 Q&A + 검증)
        │        └ 🔴 필수 수정 발견 시 해당 에이전트 재작업 (최대 2회)
        │
Phase 3  hwpx 변환 (양식 제공 시) → 07_final.hwpx
        │
Phase 4  최종 보고 (산출물 요약을 사용자에게 전달)
```

### 산출물 (`_workspace/` 디렉토리)

| 파일 | 내용 |
|------|------|
| `00_input.md` | 사용자 요구사항 정리 |
| `01_research.md` | AX 트렌드·유사 행사 벤치마크·강사/프로그램 후보 |
| `02_planning_strategy.md` | 기획 전략 (프로그램 구성, 실행가능성 분석, 리스크) |
| `03_visual_elements.md` | 표·도식 (추진체계도, 시간표, 예산표, 일정) |
| `04_planning_report.md` | **최종 기획 보고서** (개조식·공공기관 어투) |
| `05_report_commentary.md` | **작성 의미 해설** (항목별 작성 의도·논리 — 학습용) |
| `06_executive_qa.md` | **상급자 예상 질의 Q&A** + 검증 보고 |
| `07_final.hwpx` | 한글 양식 적용 최종 파일 (양식 제공 시) |

## 4. 만든 방법

1. **베이스**: [revfactory/harness-100](https://github.com/revfactory/harness-100)의 `ko/82-report-generator`(범용 업무 보고서 생성 하네스)를 클론
2. **도메인 특화**: 데이터수집→분석→시각화→집필→요약의 범용 파이프라인을 AX 행사·교육 기획 도메인으로 전면 재작성
   - 데이터 수집가 → AX 트렌드·벤치마크 리서처
   - 데이터 분석가 → 기획 전략가 (실행가능성·리스크 중심)
   - 시각화 전문가 → 기획서 표·도식 설계 (한글 문서 재현 전제)
   - 보고서 작성자 → 개조식·공공기관 어투 집필 + 해설 병행
   - 요약 검증자 → 상급자 예상 질의 Q&A + 결재 관점 검증
3. **스킬 교체**: 범용 시각화/KPI 스킬을 개조식 문체 가이드(`public-report-style`)와 상급자 질의 패턴(`executive-qa-patterns`)으로 교체
4. **hwpx 통합**: 별도 제작된 `hwpx-autofill-conversion` 스킬(hwpx ZIP·XML 구조 분석 및 양식 자동 채움)을 하네스에 통합하고 최종 산출 단계로 연결

## 5. 사용 방법

### 설치

```bash
# 1. Claude Code 설치 (https://claude.com/claude-code) 후 로그인

# 2. 이 저장소를 클론
git clone https://github.com/rocheol2/haness_ax_planning.git
cd haness_ax_planning

# 3. 클론한 폴더에서 Claude Code 실행
claude
```

VSCode 사용 시: 클론한 폴더를 VSCode로 열고 Claude Code 확장을 실행하면 `.claude/`의 에이전트·스킬이 자동 인식된다.

### 실행 예시

**풀 파이프라인** (기획 전체):

```
사내 AX 담당자 대상 생성형 AI 활용 워크샵 기획 보고서 만들어줘.
11월 개최, 50명 규모, 예산 2천만원이야.
```

**hwpx 변환 포함** (한글 양식 파일을 작업 폴더에 넣고):

```
AX 포럼 기획 보고서를 만들어서 template.hwpx 양식에 맞춰 한글 파일로 만들어줘.
```

**부분 실행**:

| 요청 예시 | 동작 |
|----------|------|
| "이 초안으로 기획서 완성해줘" | 초안 기반 집필 모드 |
| "이 기획서 개조식으로 바꿔줘" | 문체 변환 모드 |
| "이 기획서 검토해줘 / 예상 질문 뽑아줘" | 상급자 관점 검증 모드 |
| "이 보고서 hwpx로 만들어줘" (양식 첨부) | hwpx 변환 단독 모드 |

### 입력 시 알려주면 좋은 정보

- 행사 유형 (포럼 / 워크샵 / 교육 / 세미나)
- 목적, 대상과 규모, 개최 시기, 가용 예산
- 한글(hwpx) 양식 파일 (기관 표준 서식이 있는 경우)
- 기존 자료 (이전 행사 결과보고, 수요조사, 초안 등 — 있으면 해당 단계를 건너뜀)

예산·시기가 미정이어도 소/중/대 시나리오로 설계해 주므로 그대로 요청해도 된다.
